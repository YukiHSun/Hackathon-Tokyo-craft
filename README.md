# Iris Phenotype Vision Generator

This project was built for the [Gemini 3 Tokyo Hackathon](https://cerebralvalley.ai/e/gemini-3-tokyo-hackathon). It transforms the classic Iris dataset into a challenging, visual 10-round quiz game powered entirely by Google's generative AI models.

### Philosophy / プロジェクトの哲学

When learning machine learning, the "Iris" dataset is almost always the first encounter. However, merely looking at rows of numbers in a table doesn't make it easy to intuitively understand how these flowers actually look or how they differ by species. "Can we bring the world of numbers closer to human perception?" This question was the starting point of this project.

In this work, we leverage Gemini 3.1 Pro and Imagen 4.0 to transform cold feature data into visual floral forms. Differences that are hard to grasp through numbers alone become intuitive when presented as images, allowing players to think, "This might be this species because the petals are longer."

Here, AI acts not as a final answer but as a translator that extends human understanding. By bridging the abstract structure of data with visual recognition—a field where humans naturally excel—it enriches the learning experience itself. Furthermore, by placing genuine images alongside intentionally deceptive ones, players are required to do more than just "see"—they must "think, compare, and infer." The process of finding alignment between numbers and visuals becomes a uniquely human experience augmented by AI.

Challenging the abstract existence of data and turning it into something that engages intuition and reasoning—that is the ambition of this project.

## Overview / 概要

**◆ Game Overview / ゲームの概要**

The "Iris Dataset" is a familiar staple in machine learning, typically used to train AI for classification. This project flips that script. Relying solely on the flower's measurement data (sepal/petal length and width), players must infer which of the three AI-generated images is the "real" individual—the one that correctly depicts both the dimensions and the species. It is a demanding quiz game that tests human deduction against AI-generated deception.

機械学習でおなじみの「Iris Dataset（アヤメのデータセット）」。通常はAIに分類させますが、本作はその構図を逆転させました。提示される花の寸法データ（ガク・花弁の長さと幅）だけを頼りに、AIが自動生成した3枚の画像の候補から「寸法と品種を正しく描写した本物の個体1枚」を人間が推理する難解なクイズゲームです。

**◆ Game Loop / ゲームの進め方**

1. **Phase 1: Observation / 学習フェーズ**: Learn the physical characteristics of the three Iris species (Setosa, Versicolor, Virginica) using real botanical reference photographs.
   （実在する植物の参考写真を使って、3つのアヤメの品種の特徴を学習します。）
2. **Phase 2: Data Inquiry / 出題フェーズ**: In each round, you are given the specific measurements of an individual flower. **Note: All 3 candidate images are entirely AI-generated by Gemini 3.1 Pro and Imagen 4.0.**
   （各ラウンドで、ある個体の寸法と探すべき品種が提示されます。※提示される3つの候補画像は、すべてAIによって自動生成された画像です。）
3. **Phase 3: Deduction / 推理フェーズ**: Identify the scientifically accurate image among the three. The deceptive images are technically perfect generations that match the data but follow the wrong species' visual style.
   （3枚のマクロ写真から、寸法と品種を正確に表現した1枚を見抜きます。偽装画像は「別の品種」の姿を模した巧妙な仕上がりとなっています。）
4. **Phase 4: Feedback / 解析フェーズ**: Receive scoring and specialized feedback based on why a deceptive image might have fooled you, helping you refine your botanical intuition.
   （正誤判定と、なぜその画像が偽装だったのかという解説を確認し、次のラウンドへ進みます。）

## Architecture / アーキテクチャ

This project leverages two distinct AI models working in sequence to create the game assets:
1. **Gemini 3.1 Pro (`gemini-pro-latest`)**: Acting as a "Botanical Expert", it samples real statistical data from the Iris dataset and generates three distinct, highly detailed visual prompts (one accurate, two deceptive).
2. **Imagen 4.0 (`imagen-4.0-fast-generate-001`)**: Interprets the prompts to produce photorealistic, text-free images.

The frontend is a lightweight Single Page Application (SPA) built with **Vanilla JavaScript** and **TailwindCSS**.

本作は、2つのAIモデルを連携させてゲームアセットを生成しています：
1. **Gemini 3.1 Pro**: 「植物学のエキスパート」として、データセットから数値をサンプリングし、正確な描写と巧妙な偽装のための詳細なプロンプトを生成します。
2. **Imagen 4.0**: プロンプトを解釈し、フォトリアルかつ文字情報の含まれない高精度な画像を生成します。


## Setup / セットアップ

### ゲームをプレイする（推奨: Streamlit）

**Python 3.10 以上** と **Streamlit** でアプリとして起動します。venv / conda / システム Python のいずれでも構いません。

1. **リポジトリのクローン**:
   ```bash
   git clone https://github.com/YukiHSun/Hackathon-Tokyo-craft.git
   cd Hackathon-Tokyo-craft
   ```

2. **仮想環境の作成と有効化**（推奨）:
   ```bash
   python -m venv .venv
   source .venv/bin/activate   # Windows: .venv\Scripts\activate
   ```
   または conda を使う場合: `conda create -n myenv python=3.12` のあと `conda activate myenv`

3. **依存関係のインストール**:
   ```bash
   pip install -r requirements.txt
   ```

4. **アプリの起動**:
   ```bash
   streamlit run app.py
   ```

5. ブラウザで **http://localhost:8501** を開くと、Iris Phenotype Vision Generator が表示されます。

**クイズ画像について**: 画像は `app.py` が `static/assets` 以下の **.webp** を base64 で埋め込んで表示します。画像を表示するには、`static/assets/round_1/` ～ `round_10/` に各ラウンド用の .webp ファイルを配置してください。

**Streamlit Cloud でデプロイ**: このリポジトリを [Streamlit Community Cloud](https://share.streamlit.io/) に接続すると、クラウド上でアプリを公開できます。

---

### データ・画像を自前で生成する場合（任意）

新しいゲームデータや画像をローカルで生成する手順です。

1. **環境変数**: `.env` に Google AI Studio の API キーを設定します。
   ```env
   GEMINI_API_KEY=your_api_key_here
   ```

2. **依存関係**: 上記の `pip install -r requirements.txt` で、データ生成に必要なパッケージ（pandas, google-generativeai, python-dotenv）も入ります。

3. **データの生成**:
   ```bash
   python iris_server.py   # プロンプト用 JSON の生成
   ```
   画像の再生成用スクリプトがある場合は、その手順に従って実行してください。

4. **プレイ方法**: 通常は上記の **Streamlit** で起動してプレイしてください。`index.html` をブラウザで直接開くこともできますが、その場合は `app.py` による画像の埋め込みがないため、クイズ画像は表示されません。
